# AI + QA Workshop

Репозиторий для воркшопа по применению AI в тестировании.

## План воркшопа

1. **Проектирование «памяти» AI** — настройка Claude (`CLAUDE.md` и др.). Как передать контекст проекта без потери качества. Обсудим промптинг.

2. **От требований к тестам** — автоматизируем анализ спецификаций и генерируем качественные ручные тест-кейсы.

3. **API Test Automation** — ускоряем написание тестов с помощью AI.

4. **UI & L10n** — применение AI для сравнения скриншотов: проверка UI и мультиязычных интерфейсов.

## Кому будет полезно

QA-инженерам (Manual & Auto), желающим ускорить ежедневные задачи.

## Формат и подготовка

Воркшоп ориентирован на работу с Claude Code, но представленные методики и промпты адаптивны для любой среды.

**Что понадобится:**
- Ноутбук с установленной IntelliJ IDEA
- Плагин Cline или Roo Code (для поддержки `.mdc` инструкций)
- Активный API-ключ Claude (или другого AI-агента) либо доступ к веб-версии сервиса

## Оркестрация контекста: сравнение экосистем

На воркшопе мы используем структуру для Claude Code, так как она предлагает наиболее чистую реализацию "агентных навыков". Однако этот подход легко адаптируется под другие корпоративные стандарты.

### Сводная таблица конфигураций

| Инструмент | Где лежат "мозги" | Тип оркестрации | Как это работает |
|------------|-------------------|-----------------|------------------|
| **Claude Code** | `.claude/skills/*/SKILL.md` | Agentic (Explicit) | Агент сканирует папку навыков. Вы вызываете `/skill-name` или описываете задачу — агент сам подгружает нужный файл инструкций |
| **Cursor** | `.cursor/rules/*.mdc` | Contextual (Implicit) | Правила привязываются к типам файлов через glob паттерны. Активируются автоматически при открытии файла |
| **GitHub Copilot** | `.github/copilot-instructions.md` | Global + Agentic | `instructions.md` — глобальный контекст (читается всегда). Agent Skills позволяют подключать внешние инструменты |
| **OpenCode** | `.opencode/skills/*.md` | Agentic (Hybrid) | Похож на Claude Code. Позволяет определять навыки для решения сложных задач |

### Детальный разбор адаптации

#### 1. Claude Code (наш выбор)

**Философия:** "Я — автономный инженер"

**Реализация:** Навыки — отдельные Markdown-файлы с YAML-заголовком. Агент "видит" их описание и решает, какой инструмент применить.

**Плюс:** Идеальная изоляция контекста. Агент не думает про API-тесты, пока вы не попросите их написать.

#### 2. Cursor (правила .mdc)

**Философия:** "Я — умный редактор кода"

**Адаптация:** Переименуйте файлы из `skills/` в `.cursor/rules/` и добавьте заголовок `globs: *.kt`.

**Нюанс:** Cursor лучше работает с контекстом (как писать код), чем с процессом (сходи проанализируй, потом создай файл). Вы сами управляете процессом, а AI помогает внутри файла.

#### 3. GitHub Copilot (Agent Mode)

**Философия:** "Я — напарник внутри IDE"

**Адаптация:**
- **Общие правила:** Положите содержимое `qa_agent.md` в `.github/copilot-instructions.md` — база, которую Copilot будет знать всегда
- **Skills:** С декабря 2025 Copilot поддерживает Agent Skills. Для переноса навыков их нужно оформить как VS Code Extensions или использовать новый формат workspace config

#### 4. OpenCode

**Философия:** "Open Source альтернатива для агентов"

**Адаптация:** Использует структуру, схожую с Claude. Скиллы описываются декларативно.

**Плюс:** Полный контроль над тем, какие файлы отправляются в LLM, без "магии" проприетарных редакторов.

### Универсальные принципы (работают везде)

Независимо от инструмента, эти 3 принципа остаются неизменными:

1. **Project Passport** (`CLAUDE.md` / `README.md`): Хорошо описанный стек, архитектура и definition of done в корне проекта — это 80% успеха генерации

2. **Role Definition:** Промпт "Ты Senior QA, который ищет баги" работает одинаково хорошо во всех моделях

3. **Separation of Concerns:** Не создавайте один гигантский "Супер-Промпт". Разбивайте на: Анализ → Планирование → Кодинг → Ревью
